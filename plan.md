1. **Remove Information Leakage in API Endpoints:** Modify the endpoints in `backend/api/emails.py`, `backend/api/tenant_config.py`, and `backend/api/calendar.py` where exceptions are caught and directly passed in `HTTPException(status_code=400, detail=str(exc))`. I will adjust this so that sensitive user prompt content or internal provider exceptions are not directly formatted into the log message string, or simply remove the direct exposure of the exception if it contains PII.
