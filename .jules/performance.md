## 2026-06-29 - Optimize SQLAlchemy N+1 fallback in exception handlers
**Learning:** When dealing with `IntegrityError` in bulk inserts in a highly concurrent environment, blindly falling back to individual insert retries (an N+1 pattern) creates severe database bottlenecks when a large batch is processed but only a few constraints are violated.
**Action:** When catching an `IntegrityError` during a bulk insert (e.g. `db.add` multiple objects in a loop), fetch the current state of the database to proactively identify which specific records caused the constraint violation. Filter out the known conflicting objects from the session (using `db.expunge(task)` if needed so they don't break subsequent commits), and perform a secondary bulk insert for the remaining non-conflicting records. Only fall back to individual sequential inserts if the secondary batch *also* fails.
