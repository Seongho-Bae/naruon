## 2024-06-13 - Memoizing React callbacks for lists
**Learning:** In React, when a parent component renders a list of memoized child components, inline event handler functions or unmemoized functions passed as props to the children will break the `React.memo` optimization. The child components will re-render even if their data hasn't changed because the function reference changes on every parent render.
**Action:** Always wrap event handler functions (like `onSelect`, `onClick`) with `useCallback` when passing them down to memoized list item components. This ensures the function reference remains stable unless its dependencies change, allowing `React.memo` to effectively prevent unnecessary re-renders.
