## 2025-02-23 - Avoid reversing DB query results when dictionary insertion order can be preserved

**Learning:** When grouping database query results in Python 3.7+, if the results are already sorted by the database, reversing them and re-sorting them in Python introduces unnecessary `O(N)` and `O(N log N)` overhead. Previously, the `get_emails` API reversed the `DESC` array to construct `grouped` dicts in `ASC` order, and then re-sorted the result `DESC` again.

**Action:** Iterate through the native database result array (`DESC`) directly. Exploit Python 3.7+ dictionary insertion order semantics to inherently preserve that `DESC` sort order. When accumulating related items that need `ASC` order (like thread messages), append them natively during the main traversal and then explicitly reverse only the accumulated lists in a fast `O(M)` in-place traversal at the end. This maintains behavioral parity and exact contract matching for frontend clients while removing the global `O(N log N)` sorting overhead.
