## 2026-06-20 - OIDC Exchange Backend Integration and Missing Tests
**Learning:** `api.auth` is tightly coupled and modifying JWT token structures directly required rewriting the backend HTTP route test (`test_auth_real.py`). The existing legacy OIDC token assertions in `test_auth_real.py` were tightly integrated with old logic where the backend directly received OIDC tokens rather than them being exchanged at the NextJS edge layer first for an HMAC session token, indicating that those tests were out of alignment with the current architecture where OIDC operates strictly through the frontend `oidc-exchange` boundary.
**Action:** Remove invalid legacy tests that attempt to send an external OIDC token directly to the backend's standard protected endpoints, as the new architecture expects all incoming OIDC tokens to first be mapped to `naruon_session` HMAC tokens via `/api/auth/session/oidc-exchange`.
