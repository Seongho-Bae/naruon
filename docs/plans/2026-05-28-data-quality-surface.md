# 2026-05-28 Data Quality Surface

## Verified gap

- `docs/plans/2026-05-24-north-star-master-spec.md` requires the Data GNB detail
  area to cover document repository, ingestion pipeline, embeddings, and quality
  checks.
- `frontend/branding` defines the Data destination as an actionable workspace,
  but `frontend/src/components/DataLayout.tsx` still showed static ingestion
  logs, hardcoded pipeline percentages, `text-embedding-3-large`, `3,072`
  dimensions, fake indexed object counts, and fake quality issue counts.
- Backend had signed WebDAV and unique-thread intent APIs, but no `/api/data`
  router to summarize source-backed repository, ingestion, embedding, and
  quality evidence.
- A production startup log on 2026-05-28 showed `DATABASE_URL` missing during
  `uvicorn main:app` import. The code intentionally has no database URL default,
  so the fix is to support explicit `.env` lookup from root/backend/operator
  locations and require `AUTH_SESSION_HMAC_SECRET` injection in Compose and
  Kubernetes rather than adding unsafe code defaults.

## External best-practice inputs checked

- OpenTelemetry treats observability as vendor-neutral traces, metrics, and logs
  generated by the application and exported to a backend. The Data surface now
  labels connector evidence separately from planned instrumentation instead of
  presenting static sync-lag or throughput claims.
  Source: https://opentelemetry.io/docs/
- OpenLineage models jobs, runs, and datasets as consistently identified
  metadata events. This phase does not implement lineage storage yet, but the
  Data API uses opaque repository/source keys and scoped connector events as the
  future lineage boundary.
  Source: https://openlineage.io/docs/
- Great Expectations frames data quality as explicit expectations against data
  with reported outcomes. The implemented quality checks are concrete
  expectation-style checks: missing thread ids, missing duplicate fingerprints,
  blank attachment content, source registry coverage, and connector signal
  coverage.
  Source: https://docs.greatexpectations.io/

## Implemented slice

- Add signed private `GET /api/data/quality-surface`.
- Derive response values from existing rows only:
  - `webdav_accounts`
  - `project_folders`
  - `emails`
  - `attachments`
  - `connector_signal_events`
- Scope reads by signed `AuthContext`:
  - tenant/admin roles with an organization claim read the organization scope;
  - members read only their owner plus organization scope;
  - connector events are additionally scoped by workspace id.
- Return read-only operational evidence:
  - `provider_write_executed=false`
  - no provider credentials
  - no raw usernames
  - no server URLs
  - no sequential `account_id`, `folder_id`, email id, or attachment id
  - configured embedding model and the actual `Vector(1536)` dimension
- Wire Data tabs to the API:
  - document repository summary
  - connector evidence list
  - ingestion pipeline stages
  - embedding collections
  - quality checks and issue counts
- Keep WebDAV writeback and unique-thread intent controls intact as separate
  intent-only actions.

## Startup hardening

- Backend settings read environment variables first, then `~/.env`, `../.env`,
  and `.env`. This lets local operators start from `backend/`, the repository
  root, or a home-level operator env file without adding code defaults.
- Docker Compose now requires `AUTH_SESSION_HMAC_SECRET` in addition to
  `POSTGRES_PASSWORD` and composes `DATABASE_URL` from explicit env values.
- Live E2E Compose and Kubernetes backend deployment now inject
  `AUTH_SESSION_HMAC_SECRET` from explicit runtime configuration.
