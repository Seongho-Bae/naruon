## 2024-06-16 - Search input semantics, native clear buttons, and redundant disabled states
**Learning:** Adding `type="search"` to `<input>` correctly tells mobile keyboards to show a "Search" button instead of "Go", but it causes WebKit browsers to automatically inject a native cancel/clear button inside the input. If the input already has a custom React clear button, they overlap and cause visual issues. In addition, using both `disabled` and `aria-disabled="true"` on a `<button>` creates a redundant state for screen readers since the native `disabled` attribute already communicates this state and removes the element from the tab sequence.
**Action:** Always append the Tailwind class `[&::-webkit-search-cancel-button]:hidden` to inputs styled as search fields with `type="search"` if using a custom clear button. Avoid using `aria-disabled` on `<button>` elements that already have the native `disabled` attribute.
